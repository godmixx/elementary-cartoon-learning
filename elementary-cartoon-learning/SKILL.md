# Elementary Cartoon Learning — 小学卡通学习网页生成器

## 概述

根据用户指定的小学学科、年级和知识点，生成一个独立的卡通风格互动学习网页（单个 HTML 文件）。
网页包含卡通动画讲解、随堂小测验、拖拽/连线游戏、学习进度追踪四大交互模块，适合小学生自主学习使用。

## 触发条件

当用户提出以下类型请求时触发：
- "帮我生成一个小学 X 年级 [学科] [知识点] 的学习网页"
- "做个卡通学习页面，讲一下 [知识点]"
- "给小朋友做个 [学科] 互动学习工具"
- "生成 [知识点] 的互动学习 HTML"
- 任何涉及小学学科知识点 + 卡通/互动/学习网页的组合请求

## 工作流程

### 第 1 步：确认需求

从用户请求中提取以下信息，缺失项主动询问：

| 参数 | 说明 | 示例 |
|------|------|------|
| 学科 | 语文/数学/英语/科学/道德与法治/美术/音乐 | 数学 |
| 年级 | 1-6 年级 | 三年级 |
| 知识点 | 具体的知识内容 | 分数的认识 |
| 附加需求 | 特殊偏好（可选） | 多加一些练习题 |

若用户未指定学科或知识点，必须询问；年级缺失时可先按通用难度生成，但应确认。

### 第 2 步：规划网页结构

根据知识点规划以下模块（全部包含）：

1. **封面区** — 网页标题 + 卡通主题形象 + 知识点名称 + "开始学习"按钮
2. **知识讲解区** — 卡通图示 + 动画演示 + 分步骤文字说明
3. **互动练习区** — 随堂小测验（选择题/填空题）+ 拖拽/连线游戏
4. **进度追踪区** — 学习进度条 + 星星/徽章奖励 + 完成总结

参考 `references/subject-guide.md` 获取各学科的知识点组织建议和内容范例。

### 第 3 步：应用卡通设计系统

加载 `references/design-system.md`，严格遵循其中的设计规范：

- **配色**：使用明亮活泼的儿童友好色板，按学科分配主色调
- **字体**：大号圆体字，正文 ≥ 18px，标题 ≥ 28px
- **圆角**：所有卡片、按钮使用大圆角（12-24px）
- **插画**：使用 CSS 绘制 + emoji + 内联 SVG 实现卡通形象，不依赖外部图片
- **动画**：CSS 动画为主，配合少量 JS 交互动画

### 第 4 步：实现交互功能

加载 `references/interaction-patterns.md`，按以下模式实现交互：

1. **卡通动画讲解** — 知识点用分步动画演示，点击"下一步"逐步展开
2. **随堂小测验** — 3-5 道题，选择/填空，即时反馈对错 + 解析
3. **拖拽/连线游戏** — 匹配题或分类题，拖拽操作 + 动画反馈
4. **学习进度追踪** — 进度条 + 星星奖励 + 完成徽章

所有交互使用原生 JavaScript 实现，不依赖外部库。

### 第 5 步：生成 HTML 文件

将完整网页写入单个 HTML 文件，保存到用户工作目录下。文件命名规则：
`[学科]_[年级]_[知识点]_学习网页.html`（如 `数学_三年级_分数的认识_学习网页.html`）

**技术要求**：
- 单文件，HTML + CSS + JS 全部内联
- 不依赖任何外部 CDN 或网络资源
- 兼容主流浏览器（Chrome/Edge/Safari/Firefox）
- 移动端响应式适配
- 页面加载即可使用，无需服务器

### 第 6 步：验证与交付

生成后检查：
- [ ] 所有按钮和交互均可正常工作
- [ ] 测验答案正确且反馈逻辑无误
- [ ] 拖拽/连线游戏在桌面和移动端均可操作
- [ ] 进度追踪正常更新
- [ ] 卡通风格一致，色彩明快
- [ ] 文字内容适合目标年级学生的理解水平

使用 `present_files` 工具将 HTML 文件呈现给用户预览。

## 关键设计原则

1. **儿童友好** — 语言简单易懂，多用比喻和生活化举例，避免学术术语
2. **视觉吸引** — 大量使用 emoji、鲜艳色彩、圆润形状，营造游戏感
3. **即时反馈** — 每个操作都有明确的视觉/听觉反馈（动画+文字提示）
4. **循序渐进** — 知识讲解从易到难，练习题分层设计
5. **正向激励** — 答对给星星，完成给徽章，答错鼓励再试，不显示"错误"字样
6. **单文件交付** — 所有内容内联，离线可用，方便分享

## 使用示例

**用户**："帮我做一个三年级数学'分数的初步认识'的卡通学习网页"

**执行流程**：
1. 确认：学科=数学，年级=三年级，知识点=分数的初步认识
2. 规划：封面 → 分数概念动画讲解 → 分数读写练习 → 拖拽拼分数游戏 → 进度总结
3. 加载 `references/design-system.md`，应用数学学科配色（蓝色系）
4. 加载 `references/interaction-patterns.md`，实现测验和拖拽游戏
5. 加载 `assets/base-template.html` 作为骨架，填充具体内容
6. 生成 `数学_三年级_分数的初步认识_学习网页.html`
7. 用 `present_files` 呈现预览
